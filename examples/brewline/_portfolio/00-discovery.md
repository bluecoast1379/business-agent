# 业务资产盘点(Discovery)

> Brewline 黄金样例(合成数据)。访谈完成于 2026-06-15,访谈对象:财务负责人、运营负责人、采购经理、技术负责人。

## 1. 公司概况

- 公司 / 品牌:Brewline(虚构)
- 行业与商业模式:精品咖啡豆 B2B 供应——向 12 家生豆供应商(庄园与贸易商)采购生豆,自建烘焙产线,按订阅计划向约 180 家咖啡馆门店供豆;收入以门店订阅 + 单次补货订单为主,账期 NET 15 / NET 30 两档。
- 核心对象:咖啡馆门店(客户)、生豆供应商、订单、账单、发票、账期、订阅计划、烘焙批次、配送。

## 2. 现有系统与 API 清单

| 系统 | 用途 | 读/写 | 认证方式(仅键名) | 文档位置 | 备注 |
|---|---|---|---|---|---|
| Brewline ERP | 门店主档、订单、发票、账期、订阅计划、回款流水镜像 | 读+写 | Bearer token:`BACKEND_URL` / `BACKEND_API_KEY` | 内部 API 手册(仓库外) | REST,分页齐全;可为集成方签发按角色收敛的专用 token(已确认) |
| Roastery OS | 烘焙排产:烘焙批次、杯测分、豆仓库存、在途生豆 | 只读 | API key:`ROASTERY_API_URL` / `ROASTERY_API_KEY` | 供应商 OpenAPI 文档 | 批次级数据可取(已确认) |
| PayFlow(支付服务商) | 门店回款收单;回款事件以 webhook 推送 ERP | 只读 | HMAC 验签:`PAYFLOW_WEBHOOK_SECRET` | 服务商公开文档 | 回款流水已镜像入 ERP,查询走 ERP 即可(已确认) |
| 客户门户 | 门店自助查看订单与配送(网页) | 只读 | 门店账号会话 | 无独立 API 文档 | 无对外 API;后续自助助手需 ERP 侧新开门店级接口(假设) |

## 3. 数据资产

### 3.1 报表

| 报表 | 内容 | 使用者 | 频率 | 生成方式(人工/自动) |
|---|---|---|---|---|
| 经营周报 | 出货量、订阅变动、回款、Top 门店 | 总经理、运营负责人 | 每周五 | 人工(运营从 ERP 导数拼表,约 2 小时) |
| 逾期账款清单 | 逾期发票、账龄分布 | 财务负责人 | 每周两次 | 人工(ERP 导出后手工筛) |
| 烘焙批次质量表 | 批次杯测分、复烘率 | 采购经理 | 每月 | 自动(Roastery OS 导出) |

### 3.2 标签与主数据

- 门店主档:订阅档位(basic / plus / pro)、账期档(NET 15 / NET 30)、区域、开店年限——覆盖率 100%(已确认);
- 供应商主档:产地、合作年限、近四季交期达成率标签——达成率标签为采购手工维护,更新滞后约一个月(已确认);
- 豆种主数据:豆种、处理法、烘焙度基线——由 Roastery OS 维护(已确认)。

### 3.3 事件流 / 日志(字段名级描述,不贴真实数据)

- PayFlow 回款 webhook(镜像入 ERP):`payment_id, invoice_id, store_id, amount, paid_at, method`;
- ERP 订单事件日志:`order_id, store_id, event(created|packed|shipped|delivered), occurred_at`;
- Roastery OS 批次事件:`batch_id, bean_lot, roast_date, cupping_score, qc_flag`。
