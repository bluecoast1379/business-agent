# 运营与巡检 Runbook:billing-assistant

> Brewline 黄金样例(合成数据)。2026-07-10 定稿;责任人以角色名登记并经各角色确认。

## 1. 运营概览

- 责任人:值班 / 周报 / 季度复核:
  - 日常值班:运营专员(工作日检查 IM 告警与 `/status`);
  - 周报:运营负责人(每周一汇总用量 / 成本 / 抽检准确率 / 告警数);
  - 季度复核:技术负责人(模型与价格)+ 财务负责人(账务口径与术语表)。
- 关键入口:/status 端点、日志位置、scheduler 任务清单:
  - `GET /status`(Bearer `GATEWAY_AUTH_TOKEN`):会话数、月成本 `costUsd`、预算状态;
  - 日志:`agent-gateway/` 进程 stdout(部署侧按运维惯例收集,登记于 `business-agent/local/`,不写入本文档);
  - scheduler 任务:`daily-billing-patrol`、`weekly-reconciliation`、`monthly-cost-review`(与 `src/agents/registry.js` 注册名一致);手动触发:`POST /jobs/<任务名>/run`。

## 3. 成本监控

- 月度预算:200 美元(来源 business-profile `budget.monthly_usd`)
- 三道线:预警(70%)/ 熔断(100%)/ 数据源(cost-tracker `/status`):数据一律以 cost-tracker 读数为准,不以 provider 账单反推;单会话另有 maxBudgetUsd=0.5 硬顶。

| 线 | 阈值 | 通知谁/渠道 | 处置动作 |
|---|---|---|---|
| 预警线 | 月成本 ≥140 美元(70%) | 运营负责人 / IM | 当日核查用量构成(交互 vs 巡检);评估是否收紧 raw 模式使用与升档条件;预测月底超支则报总经理 |
| 熔断线 | 月成本 ≥200 美元(100%) | 运营负责人 + 总经理 / IM + 邮件 | `isOverBudget` 自动拒绝新请求(仅保留 `/health`);人工决策:追加预算(总经理审批改 env)或停服至次月 |
| 单会话硬顶 | 单会话 ≥0.5 美元 | 值班运营(周报汇总) | 会话被截停属预期护栏;单周 >3 次则分析话术模式,优化提示词或工具 summary |

## 4. 运营 SOP

### 4.1 误报处理

- 触发信号:巡检告警被财务判定为误报(如「新增逾期 6 张」实为账期档录入延迟)。
- 处置步骤:1) 值班运营在告警消息下标注误报与原因;2) 判断类型——阈值不合理则调 env(如 `PATROL_OVERDUE_NEW_MAX`)后重启,规则缺陷则修诊断脚本;3) 把误报场景加进评测集防回归;4) 周报登记误报数。
- 升级条件:同类误报一周 ≥3 次 → 财务负责人与技术负责人评审规则。

### 4.2 术语表维护

- 触发信号:抽检或用户反馈发现答非所问,根因是新黑话 / 口径变化(如财务启用新账期档)。
- 处置步骤:1) 财务负责人更新 `src/knowledge/billing-assistant-glossary.md`(必要时同步 policy 文件);2) 重启服务生效(知识文件运行时读取);3) 为新术语补 ≥2 条评测样例并跑一轮;4) 在周报记录变更。
- 升级条件:口径变化涉及计费规则 → 先过财务评审再改文件。

### 4.3 模型与价格季度复核

- 触发信号:每季度首月(由技术负责人日历驱动),或 provider 宣布模型 / 价格调整。
- 处置步骤:1) 查询 provider 当前模型列表与定价页(记录查询日期);2) 重算 roadmap 成本估算表;3) 评估分档策略:默认档 / 复杂档是否换型号(改 `LLM_MODEL` / `LLM_COMPLEX_MODEL` env);4) 换档前用评测集全量回归,通过才切换;5) 更新周报口径与预算预测。
- 升级条件:重算后月成本预测超预算 20% → 报总经理调预算或收缩场景。

### 4.4 故障处置

- 触发信号:agent 无响应、回答质量骤降、预算异常消耗、巡检未按时推送。
- 处置步骤:1) 看 `GET /status` 与进程日志(先分清:进程死 / 后端超时 / 预算熔断);2) 查 provider 状态页与 ERP 可用性;3) 回滚最近一次变更(提示词 / 知识文件 / 阈值 / 换档);4) 巡检漏发用 `POST /jobs/<任务名>/run` 手动补跑;5) 故障与处置写进周报。
- 升级条件:影响财务周对账当日完成 → 立即通知财务负责人启用人工兜底流程(上线前的旧流程保留为兜底)。

## 5. 例行报告

- 周报模板(用量/成本/抽检准确率/告警数)与发送渠道:每周一由运营负责人发运营 IM 群 + 邮件抄总经理,内容:本周会话数与周活跃提问、月成本累计与预算占比(`/status` 读数)、抽检准确率(≥10% 抽检,对照 ERP)、告警数与误报数、失败样例与评测集新增、下周关注项。

## 6. 变更管理

- 阈值调整、提示词修改、工具增删的审批与回滚方式:
  - 阈值(env):值班运营提议 → 对应干系人(财务类阈值=财务负责人,成本类=运营负责人)确认 → 修改并在周报留痕;回滚=改回旧值重启;
  - 提示词 / 知识文件:财务负责人(口径)+ 技术负责人(实现)双确认 → 评测集回归通过后发布;回滚=还原文件重启;
  - 工具增删 / 写能力启用:属波次级变更——回到 `/define-tools` 与 `/harden-agent` 流程(契约更新 + 安全复审 + 评测回归),不允许绕过工作流直接改代码;第 3 波写工具启用须先完成 02-safety-review §6 阻塞项;
  - 所有变更遵循「评测集回归通过才发布」,任何变更后观察一个巡检周期。
