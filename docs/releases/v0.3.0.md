# business-agent v0.3.0

> 本正文只描述不可变 tag `v0.3.0`（commit `a3d72bb7d670daa5e0fbf29c80830acffe221bfd`）和 `business-agent@0.3.0` 包；不把 `main` 后续的 Unreleased 变更追溯进旧版本。

## 功能摘要

- **Production profile**：加入 principal / RBAC / tenant quota、统一 durable state contract、可靠工具执行、durable scheduler、OpenAI-compatible provider、隐私默认关闭的 telemetry、hash-chain audit、只读运维 dashboard 与 deterministic eval runner。
- **版本化行业模板**：提供 retail、customer-support、finance-ops starter pack；每套包括 agent、工具 policy manifest、workflow、eval JSONL 与阈值。
- **发布证据链**：把 adapter conformance、完整 Node tests、eval / template gates、CycloneDX SBOM 与 packed-artifact scan 接入统一 `npm run check`。
- **四工具适配**：该 tag 支持 Claude Code、Cursor、GitHub Copilot 与 Codex；CodeBuddy / Trae 是 tag 之后的 Unreleased 增量，不属于 v0.3.0。

## 安全变化

- Production 启动对空 principal、memory state 与 local scheduler fail closed；工具 policy 缺项、state checksum / migration 异常及 dashboard 越权同样拒绝继续。
- Anthropic 与 OpenAI-compatible 响应具有 4 MiB 上限、可取消读取和统一 usage / stop / tool-call 契约；production 要求模型精确价格条目，避免未知模型使用默认价低估成本。
- Telemetry 默认关闭；显式启用时只导出 allowlist 脱敏元数据。Dashboard 只读，不会隐式开启采集。
- 发布包包含 SBOM，并对 packed artifact 执行内容、凭证形态与 allowlist 检查。

## 安装

前置条件：Node.js 22 或更高版本。

```bash
npm install --global business-agent@0.3.0
business-agent-init --target . --tools claude,cursor,copilot,codex --yes
```

免全局安装：

```bash
npx --yes --package=business-agent@0.3.0 business-agent-init --target . --tools claude --yes
```

## 验证

验证 Registry 与安装产物：

```bash
npm view business-agent@0.3.0 version
node -e "for (const f of ['business-agent/core/command-manifest.yaml','business-agent/scaffold','AGENTS.md']) require('node:fs').accessSync(f); console.log('business-agent ready')"
```

从 tag 复验完整 gate：

```bash
git clone --branch v0.3.0 --depth 1 https://github.com/bluecoast1379/business-agent.git
cd business-agent
npm run release:verify
```

预期：Registry 返回 `0.3.0`；初始化后的目标目录包含 workflow core、scaffold、根 `AGENTS.md` 和所选工具入口；源码复验全部 gate 退出码为 0。

## 证据与适用边界

- 仓库验证覆盖 deterministic mock、contract 和 loopback / in-process 路径；不应解读为任何客户部署、外部 provider、真实数据库、生产 ingress 或多主机存储已经获得生产认证。
- 内置 file adapter 的并发边界为同主机多进程，不覆盖跨主机或共享网络文件系统；跨节点部署需要接入通过 conformance tests 的事务 / CAS driver。
- 四个 adapter 在仓库内通过结构与一致性检查，但 `generated` / `via-AGENTS.md` 不等于在每个真实客户端版本完成人工认证。正式启用前请按 tag 中的支持矩阵人工验收。
- 示例公司 Brewline 及其数据均为合成样例，不是业务效果或客户采用证明。
