# /plan-roadmap

## Goal

把机会矩阵编译成**分波实施路线图**:按「数据就绪度 × 痛点强度 × 风险」把入选机会排进 3~4 个波次,每波带量化验收指标、成本估算方法与风险应对。产出 `agents/_portfolio/02-roadmap.md`,让投入节奏与止损点在写第一行代码前就定下来。

## Required Inputs

- `agents/_portfolio/01-opportunity-matrix.md`(前置产物,缺失则先执行 `/map-opportunities`)
- `agents/_portfolio/00-discovery.md`(核对数据就绪度结论)
- `business-agent/business-profile.yaml`(预算策略、模型 provider)
- `business-agent/core/templates/roadmap.template.md`
- `business-agent/core/knowledge/cost-model.md`(成本估算方法)

## Execution Rules

- **波次排布铁律:只读查询能力先行,写操作能力(必须人工确认)排最后一波或独立成波**。任何把 write 能力排进第一波的方案都必须向用户显式说明理由并获得确认,默认拒绝。
- 排序依据 = 机会矩阵评分:数据就绪度高 + 痛点强度高 + 改造量低的先做;三维冲突时,**数据就绪度一票优先**(数据没就绪的机会做了也是空转)。
- 建议 3~4 波;每波规模以「一个团队 2~6 周能交付」为度,首波刻意求小(1 个 agent、纯只读、单渠道),目标是快速建立信任与真实用量数据。
- **每波必须给量化验收指标**:如「目标用户周活跃提问 ≥ N 次」「答案抽检准确率 ≥ X%」「单次会话平均成本 ≤ Y 美元」;指标没有数值就不算指标。
- **成本估算给方法不给报价**:按「场景 × 日均调用次数 × 每次调用 token 量(输入/输出分开)× 模型档位」列估算表,单价留 `<TODO: 查询 provider 当前定价>` 并提示用户以 provider 官方价目为准——模型价格变动频繁,写死数字必然过期。
- **风险与应对表覆盖五类模板**(可增不可减):网络与依赖稳定性、模型幻觉、预算超支、数据安全、组织过度依赖。每类给触发信号与应对动作。
- 每波给出止损条件(什么信号出现就暂停该波并复盘),避免沉没成本绑架。

## Exit Criteria

- [ ] 波次为 3~4 波,写操作能力不在首波;若有例外已获用户显式确认并记录理由。
- [ ] 每波验收指标全部量化(有数值、有测量方法与时点)。
- [ ] 成本估算表完整,给出算式与估算方法,单价以 `<TODO: 查询当前定价>` 占位并提示用户核实。
- [ ] 风险表覆盖五类模板且每类有触发信号与应对。
- [ ] 每波有止损条件;首个 agent 的 slug 与定位已与用户确认。
